# Knowledge Articles

## Phishing Email Triage SOP

Purpose:
Standard process for reviewing user-reported phishing emails.

Steps:
1. Confirm whether the user clicked links or submitted credentials.
2. Review sender, subject, urgency language, links, and attachments.
3. Document affected user, mailbox, sender, and subject.
4. Escalate to SOC if credential exposure is suspected.
5. Recommend password reset if needed.
6. Block sender/domain if confirmed malicious.
7. Close the ticket with user guidance and evidence.

## Vulnerability Remediation and Change Request SOP

Purpose:
Standard process for tracking vulnerabilities from detection to remediation.

Steps:
1. Confirm affected asset and vulnerability severity.
2. Assign the ticket to the correct remediation team.
3. Prioritize based on asset criticality and exposure.
4. Create a change request if patching may affect availability.
5. Document implementation, test, and backout plans.
6. Apply patch during approved maintenance window.
7. Validate remediation.
8. Close the incident with evidence.
