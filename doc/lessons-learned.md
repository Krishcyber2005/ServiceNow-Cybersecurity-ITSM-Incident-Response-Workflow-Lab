# Lessons Learned

This project helped me understand how cybersecurity operations depend on structured ITSM workflows.

Key lessons:
- Security alerts need proper documentation, ownership, and follow-up.
- CMDB assets help connect incidents to affected systems.
- Assignment groups improve routing between SOC, IAM, Infrastructure, and Cloud teams.
- Vulnerability remediation should be tracked through change management when patching may affect systems.
- Work notes and closure evidence are important for auditability.
- Reporting helps security teams see workload distribution and incident trends.

Overall, this lab showed that detection is only one part of cybersecurity operations. Analysts also need to manage triage, escalation, remediation, documentation, and reporting.
