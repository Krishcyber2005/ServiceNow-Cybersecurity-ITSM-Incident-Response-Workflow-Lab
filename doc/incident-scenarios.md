# Incident Scenarios

## Scenario 1: Phishing Email Report

A user reported a suspicious Microsoft 365 login email. The incident was linked to M365-TENANT01 and assigned to Cyber Lab - SOC Tier 1.

Key actions:
- Reviewed phishing indicators
- Documented SOC triage notes
- Confirmed user guidance
- Tracked incident using ServiceNow work notes

## Scenario 2: Suspicious Login Alert

A simulated SIEM alert showed multiple failed login attempts against DC01. The incident was prioritized as critical and escalated from SOC Tier 1 to the IAM Security Team.

Key actions:
- Linked incident to DC01
- Set priority to Critical
- Documented SOC triage
- Escalated to IAM for account review

## Scenario 3: Vulnerability Remediation

A simulated vulnerability scan identified a missing security patch on WIN10-CLIENT01. The incident was assigned to Infrastructure Security and linked to a normal change request.

Key actions:
- Linked incident to WIN10-CLIENT01
- Documented risk assessment
- Created change request for patch remediation
- Added implementation, backout, and test plans
